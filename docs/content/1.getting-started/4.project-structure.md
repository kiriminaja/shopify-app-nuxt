## Overview

A `shopify-nuxt` app is a standard Nuxt application with the `shopify-nuxt` module enabled. The module auto-registers server utilities, client composables, components, OAuth routes, and App Bridge — so your project stays clean.

## Typical structure

```bash
my-shopify-app/
├── nuxt.config.ts              # Module config (apiKey, scopes, navLinks)
├── shopify.app.toml            # Shopify CLI app config
├── app.vue                     # Root component
├── layouts/
│   └── default.vue             # Wrap with <ShopifyAppProvider>
├── pages/
│   ├── index.vue               # App homepage
│   ├── products.vue            # Example page
│   └── settings.vue            # Example page
├── server/
│   ├── plugins/
│   │   └── shopify.ts          # configureShopify() — session storage, hooks
│   └── api/
│       ├── shop.ts             # Example: useShopifyAdmin(event)
│       └── webhooks.ts         # Example: useShopifyWebhook(event)
└── package.json
```

## Key files

### `nuxt.config.ts`

This is where you configure the `shopify-nuxt` module — API credentials, scopes, navigation links, and other options.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['shopify-nuxt'],

  shopify: {
    apiKey: process.env.SHOPIFY_API_KEY,
    apiSecretKey: process.env.SHOPIFY_API_SECRET_KEY,
    appUrl: process.env.SHOPIFY_APP_URL,
    navLinks: [
      { label: 'Home', href: '/', rel: 'home' },
      { label: 'Products', href: '/products' },
      { label: 'Settings', href: '/settings' }
    ]
  }
})
```

See [Configuration](/getting-started/configuration) for the full list of module options.

### `server/plugins/shopify.ts`

Use a Nitro server plugin to configure session storage, lifecycle hooks, webhooks, and billing. Simple values go in `nuxt.config.ts` — complex runtime objects go here.

```ts [server/plugins/shopify.ts]
import { configureShopify } from '#shopify/server'
import { PrismaSessionStorage } from '@shopify/shopify-app-session-storage-prisma'
import { prisma } from '~/server/utils/prisma'

export default defineNitroPlugin(() => {
  configureShopify({
    sessionStorage: new PrismaSessionStorage(prisma),
    hooks: {
      afterAuth: async ({ session, admin }) => {
        // Register webhooks, seed data, etc.
      }
    }
  })
})
```

### `layouts/default.vue`

Wrap your app pages with `<ShopifyAppProvider>` to render the App Bridge navigation menu:

```vue [layouts/default.vue]
<template>
  <ShopifyAppProvider>
    <slot />
  </ShopifyAppProvider>
</template>
```

### `server/api/*.ts`

Server API routes use the auto-imported server utilities to authenticate requests:

```ts [server/api/shop.ts]
export default defineEventHandler(async (event) => {
  const { admin } = await useShopifyAdmin(event)

  const response = await admin.graphql(`{
    shop { name currencyCode }
  }`)

  return await response.json()
})
```

## What the module registers automatically

You don't need to manually set up any of these — the module handles them:

| What                        | Description                                                             |
| --------------------------- | ----------------------------------------------------------------------- |
| **OAuth routes**            | `/_shopify/auth`, `/_shopify/auth/callback`, exit-iframe, session-token |
| **Server utilities**        | `useShopifyAdmin`, `useShopifyWebhook`, and more — auto-imported        |
| **Client composables**      | `useAppBridge()`, `useShopifyFetch()` — auto-imported                   |
| **Components**              | `ShopifyAppProvider`, `Sh*` Polaris wrappers — auto-registered          |
| **App Bridge**              | CDN script + meta tag injected via SSR head                             |
| **Polaris**                 | CDN script + web component types registered                             |
| **Auth middleware**         | `shopify-auth` route middleware (opt-in, not global)                    |
| **Default session storage** | `MemorySessionStorage` (override via `configureShopify()`)              |
