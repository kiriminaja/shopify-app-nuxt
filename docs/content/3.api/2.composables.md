## `useAppBridge()`

Returns the typed `ShopifyGlobal` from the App Bridge CDN. Auto-imported in your Vue components.

### Signature

```ts
function useAppBridge(): ShopifyGlobal
```

### Behavior

- **Safe to call anywhere** â€” returns a lazy proxy that defers property access
- **Server-side**: Returns a proxy that throws with a helpful message when any property is accessed
- **Client-side (outside iframe)**: Returns a proxy that throws when any property is accessed
- **Client-side (inside Shopify Admin)**: Returns the real `window.shopify` object

### Example

```vue
<script setup>
const shopify = useAppBridge()

onMounted(async () => {
  // Show a toast
  shopify.toast.show('Hello from App Bridge!')

  // Get a session token
  const token = await shopify.idToken()

  // Open a resource picker
  const selected = await shopify.resourcePicker({ type: 'product' })
})
</script>
```

### Available APIs

The `ShopifyGlobal` type provides access to all [App Bridge APIs](https://shopify.dev/docs/api/app-bridge-library):

| API                      | Description                      |
| ------------------------ | -------------------------------- |
| `shopify.toast`          | Show toast notifications         |
| `shopify.resourcePicker` | Open product/collection pickers  |
| `shopify.idToken()`      | Get a session token              |
| `shopify.loading()`      | Control the admin loading bar    |
| `shopify.modal`          | Show/hide App Bridge modals      |
| `shopify.config`         | App configuration (shop, locale) |

## `useShopifyFetch()`

Fetch wrapper that automatically includes the Shopify session token in the `Authorization` header. Works on both client and server.

### Signature

```ts
async function useShopifyFetch<T>(
  url: string,
  options?: RequestInit & {
    method?: string
    query?: Record<string, string | number | boolean | null | undefined>
  }
): Promise<{ data: T; response: Response }>
```

### Parameters

| Parameter | Type          | Description                              |
| --------- | ------------- | ---------------------------------------- |
| `url`     | `string`      | The API endpoint URL (e.g., `/api/shop`) |
| `options` | `RequestInit` | Standard fetch options + `query` helper  |

### Return value

| Property   | Type       | Description                    |
| ---------- | ---------- | ------------------------------ |
| `data`     | `T`        | Parsed response (JSON or text) |
| `response` | `Response` | The raw `Response` object      |

### Behavior

- **Client-side**: Fetches a session token via `shopify.idToken()` and sets `Authorization: Bearer <token>`
- **Server-side**: Forwards the incoming request's `Authorization` header

### Examples

**Basic GET:**

```vue
<script setup>
const { data } = await useAsyncData(
  'shop',
  () => useShopifyFetch('/api/shop'),
  { server: false }
)
</script>
```

**With TypeScript generics:**

```vue
<script setup lang="ts">
interface ShopData {
  shop: { name: string; currencyCode: string }
}

const { data } = await useAsyncData(
  'shop',
  () => useShopifyFetch<ShopData>('/api/shop'),
  { server: false }
)
</script>
```

**With query parameters:**

```ts
const { data } = await useShopifyFetch('/api/products', {
  query: { limit: 10, status: 'active' }
})
```

**POST request:**

```ts
const { data } = await useShopifyFetch('/api/products', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ title: 'New Product' })
})
```

::warning
Always use `server: false` with `useShopifyFetch()` in `useAsyncData` â€” session tokens are only available on the client side within the Shopify admin iframe.
::
